# CoARA Formulario — Contexto del proyecto ## Descripción Sistema de edición de fichas institucionales para el mapping de prácticas de evaluación académica del **CoARA Spanish National Chapter**. Permite a cada institución ver, editar y completar su ficha sin necesidad de rellenar un formulario desde cero. --- ## Arquitectura | Capa | Tecnología | Ubicación | |---|---|---| | Frontend | HTML/CSS/JS estático | GitHub Pages: `adelvaz.github.io/coara/formulario_coara.html` | | Backend | Google Apps Script (Web App) | Ver URL abajo | | Base de datos | Google Sheets | Importado desde `fichas_coara.csv` — hoja llamada `Datos` | **Apps Script URL:** ``` https://script.google.com/macros/s/AKfycbwN5NrqUxfys-s3slZXAfk0asRey5Hp87JFo23U0Q3sTP01x3T9P5PIHs8VKjfU5KzpnA/exec ``` **GitHub repo:** `adelvaz/coara` --- ## Flujo de datos 1. Al cargar el HTML → GET `?accion=listar` → devuelve lista de instituciones 2. Usuario selecciona institución → GET `?accion=obtener&institucion=X` → devuelve fila completa como JSON 3. Usuario edita y guarda → GET `?accion=guardar&datos={...}` con `mode: 'no-cors'` > **Nota importante:** El guardado usa GET con `mode: 'no-cors'` para evitar el bloqueo CORS de Google Apps Script desde GitHub Pages. Como consecuencia, no se puede leer la respuesta del servidor — se asume éxito si no hay excepción. --- ## Campos del formulario | Campo | Tipo de input | Estado | |---|---|---| | Institución | Texto | Existente — es el identificador único de cada fila | | Nombre de la iniciativa | Texto | Existente | | Periodo de implementación | Texto | Existente | | Enlace | URL | Existente | | Archivo_origen | Texto | Existente | | Gobernanza | Textarea | Existente | | Descripción de la acción | Textarea | Existente | | Ámbitos de evaluación | Textarea | Existente | | Metodologías y criterios | Textarea | Existente | | Recursos y financiación | Textarea | Existente | | Agente | Radio (selección única) | Nuevo | | Tipo de organización | Radio (selección única) | Nuevo | | Objeto de evaluación | Radio (selección única) | Nuevo | | Dimensiones de evaluación | Checkbox (selección múltiple) | Nuevo | ### Opciones de los campos nuevos **Agente:** Directivo/a · Evaluador/A · Técnico/a · Evaluado/a **Tipo de organización:** Universidad · Departamento o Instituto · Centro de Investigación · Agencia de Evaluación · Agencia de Financiación **Objeto de evaluación:** CV individual · Grupo de investigación · Institución · Proyecto de investigación · Política **Dimensiones de evaluación:** Investigación · Docencia · Formación · Liderazgo *(se guarda como string separado por comas, ej: `"Investigación, Docencia"`)* --- ## Estado actual del proyecto - ✅ Formulario operativo: carga y guarda correctamente - ✅ Selector de institución con carga automática de datos - ✅ Campos nuevos implementados (radios y checkboxes) - ⏳ **Pendiente:** el mensaje de confirmación de guardado aparece arriba del todo (bajo el selector), lejos del botón Guardar — hay que añadir un elemento `#estado-guardar` junto al botón y una función `mostrarEstadoGuardar()` en el JS --- ## Cambio pendiente: mensaje de confirmación junto al botón ### En el HTML, sustituir: ```html
